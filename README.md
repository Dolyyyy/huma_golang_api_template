# Golang API Template (Huma)

Simple and clean starter template for building HTTP APIs in Go with [Huma](https://huma.rocks/).

This repository gives you a clone-and-run baseline with:
- clear layering (`router`, `handlers`, `services`, `domain`)
- one ready endpoint: `GET /api/test`
- OpenAPI + docs auto-generated by Huma
- a small test suite

## Version Baseline

Latest versions used in this template on **March 2, 2026**:
- Go: `1.26.0`
- Huma: `v2.37.2`

## Why Huma

Huma is a strong fit for Go APIs when you want:
- typed request/response contracts
- automatic OpenAPI generation
- built-in docs and validation
- adapter support for popular routers/frameworks

## Quick Start

```bash
git clone https://github.com/Dolyyyy/huma_golang_api_template.git
cd golang_api_template
go mod tidy
go run ./cmd/api
```

By default, the server listens on `:8888`.

## Environment Variables (`.env`)

The app auto-loads a local `.env` file on startup.

Create your local env file from the example:

PowerShell:

```powershell
Copy-Item .env.example .env
```

Linux/macOS:

```bash
cp .env.example .env
```

Current variables:
- `PORT` (default: `8888`)
- `LOG_LEVEL` (`INFO`, `SUCCESS`, `WARNING`, `ERROR`, `CRITICAL`)
- `LOG_TO_FILE` (`true` / `false`)
- `LOG_FILE_PATH` (default: `logs/api.log`)
- `LOG_MAX_SIZE_MB` (default: `10`)
- `LOG_MAX_BACKUPS` (default: `5`)
- `LOG_MAX_AGE_DAYS` (default: `30`)
- `LOG_COMPRESS` (`true` enables gzip on rotated files)
- `LOG_CONSOLE_COLORS` (`true` / `false`)

## Logging

This template ships with a centralized logger module:
- timestamp automatically added to each line
- colored console logs with custom levels: `SUCCESS`, `ERROR`, `INFO`, `CRITICAL`, `WARNING`
- optional rotating file logs with size/retention/compression controls
- startup URL discovery (`localhost` + detected network/public URL when available)

Example output:

```text
2026-03-02T22:15:48+01:00 [SUCCESS] API is up and ready to listen
2026-03-02T22:15:48+01:00 [INFO] Local URL url=http://localhost:8888
2026-03-02T22:15:48+01:00 [INFO] Network URL url=http://192.168.1.24:8888
```

Test the API:

```bash
curl http://localhost:8888/api/test
```

Expected response:

```json
{
  "ok": true,
  "detail": "API is up and ready to listen."
}
```

## Writing API Tests

This template includes reusable test helpers in `internal/testutils`.

Current example test:
- `internal/server/http_test.go` tests `GET /api/test`
- it uses `testutils.PerformRequest` and `testutils.DecodeJSON`

Recommended flow for a new endpoint test:
1. Build the server with `server.New(config.Config{Port: "0"})`.
2. Execute the request through `testutils.PerformRequest` (or `PerformJSONRequest`).
3. Assert the status code.
4. Decode and assert JSON response payload.

## Project Structure

`cmd/api/main.go`
- application entrypoint
- starts HTTP server
- handles graceful shutdown

`internal/config`
- runtime configuration (ex: `PORT`)
- logging settings from env (`LOG_*`)

`internal/logger`
- centralized app logger
- colorized console output + rotating file output

`internal/server`
- HTTP server bootstrap
- Huma + router adapter setup

`internal/router`
- endpoint registration and wiring of dependencies

`internal/handlers`
- transport layer (HTTP operations)
- input/output mapping for API operations

`internal/services`
- business logic layer
- independent from transport details

`internal/domain`
- domain response/request models shared across layers

## How to Add a New Endpoint

1. Add/update domain models in `internal/domain`.
2. Add service methods in `internal/services`.
3. Create a handler in `internal/handlers` and register a Huma operation.
4. Wire dependencies in `internal/router/router.go`.
5. Add tests.
