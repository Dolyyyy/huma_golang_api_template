# Golang API Template (Huma)

Simple and clean starter template for building HTTP APIs in Go with [Huma](https://huma.rocks/).

This repository provides:
- a minimal API baseline (`GET /api/test`)
- clear project layering (`router`, `handlers`, `services`, `domain`)
- OpenAPI and docs auto-generated by Huma
- optional modules installable on demand via `templatectl`

## Related Repositories

- Template repo (this project): [Dolyyyy/huma_golang_api_template](https://github.com/Dolyyyy/huma_golang_api_template)
- Modules repo: [Dolyyyy/huma_golang_api_template_modules](https://github.com/Dolyyyy/huma_golang_api_template_modules)

## Quick Start

```bash
git clone https://github.com/Dolyyyy/huma_golang_api_template.git
cd huma_golang_api_template
go mod tidy
go run main.go
```

Server default address: `:8888`.

## Environment Variables (`.env`)

The app auto-loads `.env` on startup (`.env` is optional).

Create it from the example:

PowerShell:

```powershell
Copy-Item .env.example .env
```

Linux/macOS:

```bash
cp .env.example .env
```

Current baseline variables:
- `PORT` (default: `8888`)
- `LOG_LEVEL` (`INFO`, `SUCCESS`, `WARNING`, `ERROR`, `CRITICAL`)
- `LOG_TO_FILE` (`true` / `false`)
- `LOG_FILE_PATH` (default: `logs/api.log`)
- `LOG_MAX_SIZE_MB` (default: `10`)
- `LOG_MAX_BACKUPS` (default: `5`)
- `LOG_MAX_AGE_DAYS` (default: `30`)
- `LOG_COMPRESS` (`true` enables gzip on rotated files)
- `LOG_CONSOLE_COLORS` (`true` / `false`)

Module-specific env keys are injected by `templatectl add <module-id>`.

## Optional Modules (`templatectl`)

`templatectl` installs optional features from a separate modules repository.

Default source discovery order:
1. `--source <path-or-url>`
2. `TEMPLATECTL_MODULES_SOURCE`
3. `./huma_golang_api_template_modules`
4. `../huma_golang_api_template_modules`
5. `https://github.com/Dolyyyy/huma_golang_api_template_modules`

For `list`, when source is remote and a root-level `modules.json` exists, `templatectl` fetches it directly (no git clone).  
`add` still clones the repository to copy module files/templates.

Commands:

```bash
go run ./cmd/templatectl list
go run ./cmd/templatectl add auth-token
go run ./cmd/templatectl add metrics-prometheus
go run ./cmd/templatectl remove auth-token
go run ./cmd/templatectl doctor
```

Useful flags:
- `--source <path-or-url>`
- `--root <project-path>`
- `--skip-verify` (skip `go mod tidy` + `go test ./...`)
- `--no-color`
- `--no-spinner`

When installing/removing a module:
- `templatectl` checks that git working tree is clean (if `.git` exists)
- copies module files into the project
- updates `.env` defaults
- regenerates `internal/modules/generated_imports.go`
- updates `.templatectl.lock.json`
- runs `go mod tidy` and `go test ./...` (unless `--skip-verify`)

## Suggested Repository Strategy

- [huma_golang_api_template](https://github.com/Dolyyyy/huma_golang_api_template): minimal base + `templatectl`
- [huma_golang_api_template_modules](https://github.com/Dolyyyy/huma_golang_api_template_modules): installable module packs (`auth-token`, `metrics-prometheus`, DB modules, etc.)

This keeps clone size and dependencies small by default.
